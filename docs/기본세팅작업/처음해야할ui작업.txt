프론트엔드 초기 UI 작업 추천 순서

1. Headless UI 및 Tailwind CSS 설정
   - 디자인 자유도를 높이고, 직접 제어하는 공통 컴포넌트 기반 마련

2. 전역 레이아웃 구성 (app/layout.tsx)
   - 모든 페이지에 공통으로 적용될 헤더, 푸터 등 뼈대 제작

3. 헤더에 auth 메뉴 로그인, 회원 가입 컴퍼넌트 보이게 필요하면 zustand 설치
백엔드와 연동 해서 jwt 로그인 보편적인 로직으로 구현
   - 서비스 첫인상, 로그인/회원가입 유도

4. 회원가입 / 로그인 페이지 제작
   - 사용자 인증을 위한 폼(Form) 화면 구성

5. 핵심 기능 (업무 대시보드) 페이지 프로토타이핑
   - 업무 목록, 추가 등 핵심 기능 UI 구성
